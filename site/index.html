⚓ haw

reproducible multi-repo stacks · cross-repo PRs · one Rust binary

haw (hawser) assembles a software stack out of many independent Git repositories — no submodules, no detached HEADs, no Python runtime. A declarative haw.toml describes the stack; a committed haw.lock pins every repo to an exact revision, so any teammate or CI machine reproduces the same tree. Then it drives the day-to-day: branch a feature across every affected repo, open the linked PRs/MRs on GitHub and GitLab, and track review + CI from one keyboard-driven cockpit.
